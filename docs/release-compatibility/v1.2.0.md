# scanq-shared v1.2.0 Compatibility Guide

## Overview

v1.2.0 is a MINOR release. The change set is additive and preserves all existing v1.x
contracts.

## New contract areas

| Area | What changed | Additive? | Migration action |
|---|---|---:|---|
| Context resolve | No shape changes | Yes | None |
| Service token | No shape changes | Yes | None |
| Lineage register | No shape changes | Yes | None |
| Lineage finalize | No shape changes | Yes | None |
| Media compose | New request/response schemas and typed client method | Yes | Import the new contracts from `scanq_shared.schemas` and `scanq_shared.enums` |
| Error envelope | New typed envelope alongside existing `ErrorResponse` | Yes | Prefer `ErrorEnvelope` for new code; keep `ErrorResponse` for legacy callers |

## Stable imports

```python
from scanq_shared.clients import TrainingStudioClient
from scanq_shared.enums import MediaType, MediaComposeStatus, ErrorCode
from scanq_shared.schemas import MediaComposeRequest, MediaComposeResponse, ErrorEnvelope
```
